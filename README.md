## 📖 About the Project

This is a **production-grade end-to-end automation framework** built for the e-commerce demo website [SauceDemo](https://www.saucedemo.com/).

The framework automates **40 test scenarios** covering all critical user journeys:
- User Authentication (Login / Logout)
- Product Search and Sorting
- Shopping Cart Operations
- Complete Checkout Flow

The project was built following industry best practices including:
- **Page Object Model (POM)** design pattern for maintainability
- **Data-Driven Testing** using CSV files for maximum test coverage
- **Automatic HTML Reports** generated by TestNG after every run
- **WebDriverManager** for automatic ChromeDriver management

## 🏗️ Design Pattern

### Page Object Model (POM)

This framework uses the **Page Object Model** design pattern where each page of the application has its own Java class.

```
WITHOUT POM:                    WITH POM:
                                
Test 1: driver.findElement      LoginPage.java:
  (By.id("user-name"))            - usernameField locator
Test 2: driver.findElement        - passwordField locator  
  (By.id("user-name"))            - login() method
Test 3: driver.findElement      
  (By.id("user-name"))          Tests just call:
                                  loginPage.login(user, pass)
If ID changes = fix 3 files     If ID changes = fix 1 file
```

**Benefits:**
- Change locator in ONE place — all tests work automatically
- Tests are clean and readable — no HTML details in test code
- Easy to add new tests — just call existing page methods
- Easy to maintain — one class per page

## ▶️ How to Run

### Method 1 — Run via Eclipse (Recommended for Beginners)

```
1. Open Eclipse
2. Import project: File → Open Projects from File System
3. Right-click testng.xml
4. Click Run As → TestNG Suite
5. Watch 40 tests run automatically!
```

### Method 2 — Run via Maven Command Line

```bash
# Navigate to project root folder
cd C:\workspace\SeleniumFramework

# Run all tests
mvn test

# Run specific test class only
mvn test -Dtest=LoginTest

# Run with specific browser (future enhancement)
mvn test -Dbrowser=chrome
```

### Method 3 — Run Individual Test Class

```
1. Right-click any test file (e.g. LoginTest.java)
2. Run As → TestNG Test
3. Only that class runs
```

## 🔑 Key Concepts Used

### ThreadLocal WebDriver
```java
// Makes tests safe to run in parallel
// Each thread gets its own browser instance
protected static ThreadLocal<WebDriver> driver = new ThreadLocal<>();
```

### Implicit Wait
```java
// Waits up to 10 seconds for elements before failing
// Applied globally to all findElement() calls
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
```

### Data-Driven Testing
```java
// Reads all rows from CSV — one test runs for all data sets
List<Map<String, String>> testData = CSVReader.readCSV("credentials.csv");
for (Map<String, String> row : testData) {
    loginPage.login(row.get("username"), row.get("password"));
}
```

### Stale Element Fix
```java
// Re-finds elements after page re-renders
// Prevents StaleElementReferenceException
while (!driver.findElements(removeButtons).isEmpty()) {
    driver.findElements(removeButtons).get(0).click();
}
```
